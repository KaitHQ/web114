Why is the JavaScript textContent property safer than innerHTML?

textContent is safer because it just adds plain text into a webpage and it doesn't treat anything like HTML code. So even if someone tries to sneak in a script or some malicious code, it won't run. On the other hand, innerHTML will treat what you insert as real HTML, which means it can accidentally allow dangerous stuff to run if you're not careful. Safety is important, so for simpler tasks like adding text, textContent is the better and safer choice.

Explain the relationship between an HTML attribute and a corresponding DOM property. Use a specific example.

An HTML attribute is what you write in the HTML file, while a DOM property is how you work with that element using JavaScript once the page has loaded. For example, if you have this in your HTML: 

<input type="text" id="nameField" value="Tomira">

The value "Tomira" is the attribute. It sets the starting value. But in JavaScript, you can grab that field by means of a document.getElementById function and change its value like this: 

document.getElementById("nameField").value = "Zelda";

Changing the value property updates what appears in the input box, but doesn't affect the original HTML attribute. You can change those values in the console.
